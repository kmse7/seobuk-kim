# AI에게 전문 기술 가르치기 — 스킬과 커맨드

## 스킬이 뭔가요?

스킬은 **AI에게 특정 업무 매뉴얼을 주는 것**입니다.

### 비유: 신입사원 교육

```
신입사원: "이건 어떻게 해요?"
선배: 매번 설명하기 귀찮으니까
     "이건 이 매뉴얼 보고 해" (매뉴얼 건넴)

Claude Code도 똑같음:
/design-review라고 치면
AI가 "디자인 리뷰 전문가" 모드로 자동 변신
```

---

## 우리 회사에서 쓸 수 있는 스킬

### 1️⃣ `/design-plan` — "새 페이지 디자인 잡아줘"

### 이런 상황일 때

```
새로운 기능의 UI/UX를 처음부터 설계해야 할 때
```

### AI한테 이렇게 말하세요

```
/design-plan
사용자 프로필 페이지 새로 설계해줘.
요구사항:
- 프로필 사진 변경
- 통계 대시보드
- 월별 활동 차트
```

### AI가 해주는 것

1. 인터뷰: "사용자가 제일 먼저 봐야 할 것은?"
2. 디자인 생성: 레이아웃 스케치
3. 검토: "이 색상이 좋아 보여?"
4. 개선 제안: 버튼 위치 조정, 폰트 크기 등
5. 문서화: 최종 설계를 DESIGN.md로 저장

### ✅ 좋은 예

```
/design-plan
결제 화면 재설계해줘.
현재: 3단계 (카드입력 → 주소 → 확인)
목표: 1단계로 줄이기 + 구글페이 추가
이유: 사용자가 중간에 포기함
```

### ❌ 안 좋은 예

```
/design-plan
UI 디자인 해줘
```

---

### 2️⃣ `/design-review` — "이 화면 디자인 검토해줘"

### 이런 상황일 때

```
이미 만든 화면이 있는데,
디자인이 좋은지 검토하고 싶을 때
```

### AI한테 이렇게 말하세요

```
/design-review
[스크린샷 첨부]
이 로그인 폼 사용하기 편해 보여?
```

### AI가 해주는 것

1. 스크린샷을 분석
2. "버튼이 너무 작다", "색상 대비가 약하다" 등 지적
3. 개선 아이디어 제시
4. 코드 레벨 수정안 제공

### ✅ 좋은 예

```
/design-review
[스크린샷]
이 결제 폼, 사용자가 실수하기 쉬워 보여?
빨강색으로 경고가 띄워지는 부분이 눈에 띄어?
```

### ❌ 안 좋은 예

```
/design-review
이거 좋아?
```

---

### 3️⃣ `/review-ai` — "코드 리뷰해줘"

### 이런 상황일 때

```
코드를 작성했는데,
버그나 보안 문제가 없는지 확인하고 싶을 때
```

### AI한테 이렇게 말하세요

```
/review-ai src/auth.ts
```

### AI가 해주는 것

1. 코드 읽음
2. 버그 찾기: "null check 빠짐"
3. 보안: "비밀번호가 평문으로 저장됨"
4. 성능: "N+1 쿼리 문제"
5. 개선 코드 제시

### ✅ 좋은 예

```
/review-ai src/payment/stripe.ts
focus: 보안 (결제 데이터), 성능 (대량 주문 처리)
```

### ❌ 안 좋은 예

```
/review-ai
코드 검토해줘
```

---

### 4️⃣ `/cicd-setup` — "이 프로젝트 자동 배포 설정해줘"

### 이런 상황일 때

```
새 프로젝트를 만들었는데,
코드를 푸시하면 자동으로 테스트 → 배포되게 하고 싶을 때
```

### AI한테 이렇게 말하세요

```
/cicd-setup
새 Node.js 프로젝트 CI/CD 해줘.
테스트 → 빌드 → 배포 자동화해줘.
```

### AI가 해주는 것

1. GitHub Actions 워크플로우 생성
2. 자동 테스트 설정
3. PM2로 프로덕션 배포 설정
4. 문서 작성

### ✅ 좋은 예

```
/cicd-setup
React + Node.js 풀스택 프로젝트.
- GitHub에 푸시하면 자동 테스트
- 테스트 통과하면 자동 배포
- Slack 알림도 보내줘
```

### ❌ 안 좋은 예

```
/cicd-setup
배포 설정해줘
```

## 커맨드 — "큰 작업 자율적으로 해줘"

### 1️⃣ `/ultrawork` — 큰 작업을 한 번에

### 이런 상황일 때

```
"TypeScript 타입 에러를 다 고쳐줘"
"코드 스타일을 통일해줘"
이런 큰 작업을 자율적으로 진행하고 싶을 때
```

### AI한테 이렇게 말하세요

```
/ultrawork
TypeScript 타입 검사 0 에러로 만들어줘.
src/ 폴더 전체.
```

### AI가 해주는 것

1. 모든 타입 에러 찾기
2. 자동 수정 시도
3. 수정 안 되는 부분은 경고
4. 최종 결과 보고

### ✅ 좋은 예

```
/ultrawork
정규표현식 유효성 검사 추가해줘.
src/utils/validators.ts와 그걸 쓰는 모든 파일.
```

---

### 2️⃣ `/setup-workspace` — "여러 팀이 동시에 작업하게 해줘"

### 이런 상황일 때

```
프론트엔드, 백엔드, 테스트 팀이
한 번에 다른 부분을 수정해야 할 때
```

### AI한테 이렇게 말하세요

```
/setup-workspace
프론트엔드, 백엔드, QA 동시 작업
```

### AI가 해주는 것

1. 터미널을 여러 칸으로 분할
2. 각 칸에 에이전트 배치
3. 부분별로 동시 진행

## 나만의 스킬 만들기

### 이런 상황일 때

```
"우리 회사는 항상 커밋 메시지를 이렇게 쓴다"
"새 페이지 만들 때는 항상 이 템플릿을 쓴다"
이런 반복 작업이 많을 때
```

### 해결법: 스킬 문서 작성

프로젝트 폴더에 `.claude/skills/my-skill/SKILL.md` 파일 만들기:

```markdown
---
name: "commit-message"
description: "우리 회사 커밋 메시지 규칙"
---

# 커밋 메시지 스킬

커밋 메시지는 다음 형식으로 작성:
1. 제목: 50자 이내, 과거형
2. 본문: 72자씩 줄바꿈
3. 이슈 번호: "Fixes #123"

예시:
feat(auth): 구글 로그인 추가

구글 OAuth를 구현했습니다.
- oauth.google.com API 등록
- 로그인 버튼 추가
- 토큰 저장 로직 구현

Fixes #456
```

### 사용하기

```
/commit-message
파일 수정 내용을 커밋 메시지로 만들어줘.
```

### 💡 팁
- 자주 반복하는 "매뉴얼 같은 작업"을 스킬로 만들면 편합니다
- 팀 전체가 같은 규칙을 따르니까 혼동이 없습니다

## 플러그인 마켓플레이스 — "다른 사람이 만든 스킬 설치하기"

### 이런 상황일 때

```
다른 회사에서 만든 좋은 스킬이 있는데,
우리도 쓰고 싶을 때
```

### 설치하기

```
/plugin marketplace add
[마켓플레이스 스킬명]
```

### ⚠️ 주의
- 회사 규칙과 맞는지 확인 후 설치하세요
- 불필요한 권한을 요청하는 스킬은 피하세요

## 실전 예시

### 상황 1: "새 페이지 디자인이 필요함"

```
/design-plan
관리자 대시보드 새로 설계해줘.
현재: 표 중심
목표: 시각적 차트 + 필터 기능
```

→ AI가 디자인 검토하고 문서 작성

### 상황 2: "코드에 버그가 있을 것 같아"

```
/review-ai src/database/connection.ts
```

→ AI가 보안/성능 문제 찾음

### 상황 3: "GitHub에 푸시하면 자동 배포되게 하고 싶어"

```
/cicd-setup
Next.js + Express 백엔드 구성.
GitHub Actions + PM2 사용해줘.
```

→ CI/CD 파이프라인 자동 구성
