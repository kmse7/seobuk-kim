# AI가 자꾸 까먹을 때 — 대화 관리법

## 왜 까먹나요?

AI의 기억력에는 한계가 있습니다. 대화가 길어지면 앞부분을 잊어버려요.

**비유**: 시험공부할 때 책 앞부분을 계속 읽다 보니 처음에 뭘 배웠는지 까먹는 것처럼.

### 구체적 예시

**처음 1시간 (기억력 좋음):**
```
당신: "Python 프로젝트야"
AI: "좋아, Python으로 뭘 만드나?"
당신: "API 서버"
AI: "Python API 서버군요!"
```

**3시간 후:**
```
당신: "이 에러가 뭐야?"
AI: "음... 뭐하는 프로젝트였더라?"
당신: "아, Python으로 API 서버 만드는 거야"
AI: "아, 맞다!"
```

**8시간 후:**
```
당신: "처음 구조 말해줄 수 있어?"
AI: "흠... 뭘 하셨는지 잘 모르겠는데..."
당신: "😤"
```

---

## 기억력 상태 확인하기

### statusline으로 현재 상태 확인

```bash
/status
```

결과 예시:

```
Claude Code Status
─────────────────────────────────
프로젝트: ~/projects/myapp
모드: default
세션: main
메모리 사용: 65% ← 지금 기억력 65% 찼음
대화 길이: 523 lines
마지막 활동: 2분 전
```

**기억력 해석:**

| 기억력 | 상태 | 권장 조치 |
|-------|------|---------|
| 0-50% | 여유 있음 | 계속 진행 가능 |
| 50-75% | 조심 | 대화가 길어지지 않도록 주의 |
| 75-90% | 위험 | `/compact` 고려 |
| 90%+ | 긴급 | 지금 당장 `/compact` 또는 `/clear` |

---

## 기억력 아끼는 법 3가지

### 1. "한 번에 한 가지만" (가장 중요)

❌ **안 좋은 예:**
```
이 에러 고쳐주고, 이 기능도 추가하고, 문서도 작성해줄 수 있을까?
```
→ AI가 3가지를 동시에 처리하려다 헷갈림

✅ **좋은 예:**
```
먼저 이 에러를 고쳐줄 수 있을까?
```
(끝나면) → "다음으로 기능을 추가해줄 수 있어?"

**효과:**
- AI가 한 가지에만 집중
- 기억력 50% 절약
- 품질 향상

---

### 2. "새 주제 시작하면 /clear 하기"

새로운 프로젝트나 새로운 기능으로 넘어갈 때:

```bash
/clear
```

**타이밍:**
- 한 기능 완료했을 때
- 새 파일 작업 시작할 때
- 버그 수정 끝났을 때

**장점:**
- AI의 기억 초기화 (깨끗한 상태)
- 기억력 100% 복구
- 다음 작업에 집중 가능

**예시:**

```
당신: "회원가입 기능 만들어줄 수 있어?"
AI: (작업 중...)
당신: "좋아, 다 끝났어. /clear"

(새로 시작)
당신: "이번엔 로그인 기능 만들어줄래?"
AI: "새로운 프로젝트인가요? 처음부터 설명해주세요"
당신: "로그인 페이지와 API 백엔드"
AI: (깨끗한 상태에서 집중)
```

---

### 3. "길게 조사할 땐 subagent 사용하기"

대용량 파일을 읽거나 많은 검색이 필요할 때:

❌ **안 좋은 예:**
```
지난 100개 커밋을 분석해줄 수 있을까?
```
→ AI가 모두 기억하려다 기억력 바닥남

✅ **좋은 예:**
```
/explore "지난 100개 커밋을 분석해서 CHANGELOG.md 만들어줄래?"
```
→ Subagent가 대신 처리 (메인 세션 기억력 아낌)

**효과:**
- 메인 AI의 기억력 보호
- 복잡한 작업도 빠르게 처리
- 당신은 다른 일 계속 가능

---

## AI한테 기억시키기

### 방법 1: CLAUDE.md (업무 매뉴얼)

이것은 "모든 AI가 읽어야 하는 규칙"입니다.

**어디에 있나?**
```
프로젝트 루트/CLAUDE.md
```

**뭘 넣을까?**
```markdown
# 이 프로젝트의 규칙

## 언어
- 코드: 영어
- 주석: 한국어
- 커밋 메시지: 영어

## 패턴
- API 응답 형식: JSON
- 에러 처리: try-catch 사용
- 테스트: Jest 사용

## 금지 사항
- DB 암호 파일에 넣지 말 것
- console.log() 프로덕션 코드에 남기지 말 것
```

**효과:**
- AI가 매번 설명을 듣지 않아도 됨
- 일관된 스타일 유지
- 기억력 20% 절약

---

### 방법 2: 메모리 (AI의 메모장)

긴 프로젝트를 여러 세션에서 진행할 때, AI가 기억해야 할 정보:

**파일 위치:**
```
~/.claude/projects/[프로젝트명]/memory/
```

**예시:**

memory/project-progress.md:
```markdown
# 프로젝트 진행상황

## 완료된 것
- [x] 회원가입 기능
- [x] 로그인 API
- [x] DB 설계

## 진행 중
- [ ] 비밀번호 리셋
- [ ] 이메일 인증

## 알아야 할 것
- API 응답 형식: { code, message, data }
- DB: PostgreSQL
- 에러 코드: 400(입력 오류), 401(인증 오류), 500(서버 오류)
```

AI가 새 세션에서:
```
당신: "진행상황 어디까지?"
AI: (메모리 확인) "회원가입과 로그인은 완료했고, 비밀번호 리셋 진행 중입니다"
```

**효과:**
- 멀티세션에서도 일관성 유지
- 새 AI도 바로 상황 파악 가능
- 기억력 30% 절약

## AI가 느려지면?

### 방법 1: /compact (메모리 정리)

대화가 너무 많아져서 느려질 때:

```bash
/compact
```

**효과:**
- 지금까지의 대화를 요약
- 불필요한 부분 정리
- 기억력 70% → 40%로 감소 (여유 생김)
- 속도 2-3배 향상

**타이밍:**
- 한 기능 완료 후
- 대화가 500줄 이상일 때
- 느려지는 게 느껴질 때

---

### 방법 2: /clear (완전 초기화)

더 깨끗한 상태로 시작하고 싶을 때:

```bash
/clear
```

**효과:**
- 모든 대화 기록 삭제
- 기억력 100% 복구
- 완전히 새로운 상태

**주의:**
- /clear 전에 중요한 진행상황은 Git에 커밋하기
- 또는 메모리에 저장하기

## 💡 실전 패턴: 하루 업무 루틴

```bash
cd ~/my-project

```

**AI한테 첫 인사:**
```
안녕! 어제 작업한 로그인 기능 테스트해줄 수 있을까?
```

---

### 오전 중: 한 기능씩 처리

```
당신: "로그인 테스트 통과했어?"
AI: "네, 모두 통과했습니다"
당신: "좋아! /clear"

(새로 시작)
당신: "이번엔 비밀번호 리셋 기능을 만들어야 해"
AI: "새 프로젝트군요? 상세히 설명해주세요"
```

---

### 오후 2시: 기억력 확인

```bash
/status
```

메모리 사용이 80% 이상이면:

```bash
/compact
```

---

### 저녁 5시: 마무리

```bash
/status  # 진행상황 확인
git add . && git commit -m "feat: 비밀번호 리셋"
```

### 내일 아침: 어제 이어하기


메모리 파일 읽기:
```
어제 비밀번호 리셋까지 했어. 메모리에 저장된 진행상황 확인해줄 수 있을까?
```

## ⚠️ 주의사항

### 1. /clear는 돌이킬 수 없어

```
❌ 하면 안 됨:
/clear → 아, 잠깐! (이미 늦음)

✅ 해야 할 것:
/clear 하기 전에 git commit
또는 메모리에 저장
```

### 2. 메모리에 보안 정보 넣지 말기

```
❌ 안 됨:
memory/secrets.md:
API_KEY=sk_live_xxx
DB_PASSWORD=mypassword123

✅ 좋음:
memory/config.md:
API 사용 중
DB는 prod-db 사용
(암호는 .env 파일에만)
```

### 3. 메모리 파일도 Git에 커밋하지 말기

```bash
# .gitignore
~/.claude/projects/*/memory/
```

## 실수했을 때

### 실수 1: /clear를 잘못 해버렸어요

```bash
git log  # 최근 커밋 확인
git show HEAD  # 내용 확인
git reset --soft HEAD~1  # 필요하면 되돌리기
```

### 실수 2: AI가 계속 같은 실수를 반복해요

**해결:**
1. CLAUDE.md에 규칙 추가 (AI의 "업무 매뉴얼" 업데이트)
2. 예시 제시: "이렇게 해줘" (구체적으로)
3. /clear 후 다시 시작

### 실수 3: 멀티세션인데 어느 프로젝트 정보인지 헷갈려요

**해결:**
```bash
/status  # 현재 세션 확인
```

각 프로젝트마다 별도의 memory/ 폴더 유지
