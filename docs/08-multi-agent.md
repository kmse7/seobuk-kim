# AI 여러 명이 동시에 일하게 하기 — 멀티에이전트

## 왜 필요한가요?

혼자 일을 하면 느립니다. 여러 명이 나눠서 하면 빠릅니다. AI도 마찬가지입니다.

예를 들어:
- **혼자 하는 경우**: "이 프로젝트 구조 파악해줄래? 그리고 이 버그도 봐줄래? 그리고 문서도 작성해줄래?"
  → AI가 한 번에 하나씩만 처리 (느림)

- **여러 명이 하는 경우**: "A 인턴은 프로젝트 구조 파악해, B 인턴은 버그 찾아, C 인턴은 문서 작성해"
  → 동시에 진행 (빠름)

**Claude Code에서 이렇게 일을 나눌 수 있습니다.**

---

## 4가지 상황별 가이드

### 상황 1: "뭔가 조사해야 할 때" → Subagent

**언제 쓰나요?**
- 대용량 파일을 읽거나 검색해야 할 때
- 리서치나 문서 작성이 필요할 때
- 지금 하는 작업을 방해받고 싶지 않을 때

**비유**: 인턴에게 "이거 조사해서 정리해놔" 라고 하고, 본인은 다른 일 계속 하는 거

#### 💬 AI한테 이렇게 말하세요

```
/explore "프로젝트의 모든 API 엔드포인트를 찾아서 API.md 파일로 정리해줄래?
- 각 엔드포인트의 경로, 메서드, 설명 포함
- 예시 요청/응답도 넣어"
```

또는:

```
백그라운드에서 지난 3개월 간의 변경사항을 분석해서 CHANGELOG.md 만들어줄래?
```

#### 📝 Claude Code가 할 일

1. 별도의 백그라운드 에이전트 생성
2. 당신의 메인 세션을 방해하지 않음 (계속 다른 일 가능)
3. 조사 완료 후 파일 생성 또는 정보 제공

#### ✅ 성공하면

- API.md 또는 CHANGELOG.md 같은 새 파일 자동 생성
- 당신은 그 사이에 다른 일 계속 가능
- 완료 후 확인만 하면 됨

---

### 상황 2: "여러 파일을 동시에 수정해야 할 때" → Subagent + Worktree

**언제 쓰나요?**
- 2-5개 정도의 **독립적인** 파일을 동시에 수정할 때
- 파일끼리 영향을 주지 않을 때

**비유**: 3명의 인턴에게 각각 다른 문서를 할당하고, 동시에 작업하게 하는 것

#### 💬 AI한테 이렇게 말하세요

```
이 3개 파일을 동시에 수정해줄래?
- src/pages/login.js: 로그인 페이지 UI 개선
- src/api/auth.js: API 응답 에러 처리 추가
- src/styles/theme.css: 다크모드 지원
```

#### 📝 Claude Code가 할 일

1. 각 파일마다 별도의 "작업 공간(worktree)"을 만듦
2. AI 여러 명이 동시에 각각의 파일을 수정
3. 병합(merge) 자동화

#### ✅ 성공하면

```
✓ login.js 수정 완료
✓ auth.js 수정 완료
✓ theme.css 수정 완료
→ 모두 자동으로 병합됨
```

#### ⚠️ 주의

```
❌ 금지: 같은 파일을 여러 AI가 동시에 수정
  (충돌 발생, 데이터 손실 가능)

✅ 허용: 각각 다른 파일만 수정
```

---

### 상황 3: "프론트+백엔드+테스트 동시에 해야 할 때" → Agent Teams

**언제 쓰나요?**
- 복잡한 프로젝트 (front / back / test 다 필요)
- AI들이 서로 소통해야 할 때
- 한 AI의 작업이 다른 AI에게 영향을 줄 때

**비유**: 프로젝트팀을 구성해서, 팀장이 각자 진행 상황을 보고받고 조정하는 것

#### 💬 AI한테 이렇게 말하세요

```
팀을 꾸려서 새 기능을 만들어줄래?
- 설계자: 데이터 구조와 API 설계
- 개발자: 백엔드 코드 작성
- 테스터: 테스트 케이스 작성 및 검증
```

또는 명령어로:

```
/team-create
```

그럼 대화형 메뉴가 나옵니다.

#### 📝 Claude Code가 할 일

1. 여러 에이전트(AI) 구성
2. 각각의 역할 할당
3. 에이전트들이 서로 소통하면서 진행
4. 완성된 코드 + 문서 + 테스트 결과 제공

#### ✅ 성공하면

```
✓ 설계자가 schema.md 작성
✓ 개발자가 그걸 보고 코드 구현
✓ 테스터가 테스트 케이스 작성하고 검증
→ 모두 완료 & 문제 없음
```

---

### 상황 4: "큰 작업을 자율적으로 맡기고 싶을 때" → /ultrawork

**언제 쓰나요?**
- 매우 복잡한 작업
- 계획 → 실행 → 검증의 단계가 필요할 때
- 당신이 자세히 지시하지 않고 싶을 때

**비유**: 팀장한테 "이 프로젝트 자주 봐가면서 해결해줄래?"라고 위임하는 것

#### 💬 AI한테 이렇게 말하세요

```
/ultrawork "성능 최적화: 이미지 로딩 속도 2배 빠르게 하기. 빌드 시간도 50% 줄이기"
```

또는:

```
/ultrawork "사용자 인증 시스템 전체 리팩토링. OAuth 추가 및 보안 감시 포함"
```

#### 📝 Claude Code가 할 일

**3단계로 자동 진행됩니다:**

1. **Planning (계획)**
   - 작업을 어떻게 나눌지 분석
   - 리스크 식별
   - 테스트 계획 수립

2. **Execution (실행)**
   - 코드 작성
   - 테스트 실행
   - 오류 수정

3. **Verification (검증)**
   - 최종 테스트
   - 성능 측정
   - 문서 작성

#### ✅ 성공하면

- 계획 문서
- 완성된 코드
- 테스트 결과
- 사용 가이드

모두 한 번에 받을 수 있습니다.

## 상황 선택 판단표

| 상황 | 사용 방식 | 이유 |
|------|---------|------|
| "이 폴더 분석해줄래?" | Subagent | 한 명이 충분, 빠름 |
| "이 3개 파일 수정해줄래?" | Subagent + Worktree | 병렬 처리로 3배 빠름 |
| "앱 만들어줄래?" | Agent Teams | front/back/test 협업 필요 |
| "버그 찾아줄래, 시간 많아" | Agent Teams (3-5명) | 다각도 분석 |
| "이 큰 프로젝트 해줄래?" | /ultrawork | 자동 계획 & 검증 |

## 💡 팁 3가지

### 팁 1: Subagent는 백그라운드가 최고

```
/explore "README 작성"

→ 당신은 코드 수정 계속 가능
→ 동시에 진행됨
→ 시간 절약
```

### 팁 2: 같은 파일 금지!

```
❌ 안 됨:
worker-1이 src/auth.js 수정 중
worker-2도 src/auth.js 수정

✅ 좋음:
worker-1 → src/auth.js
worker-2 → src/api.js
worker-3 → src/utils.js
```

### 팁 3: Agent Teams는 소통이 핵심

```
architect: "API 설계 완료!"
dev: "이 설계대로 코드 짤게"
qa: "테스트 케이스 만들기"

→ 이렇게 자동으로 진행됨
```
